# פרויקט גמר: חדרי בריחה - למידה עם חיזוקים (Hizki In Space RL)

## מה הפרויקט?

בנינו משחק חדרי בריחה שבו הסוכן הוא **חיזקי** - כלב בחליפת חלל שמנסה למצוא עצם בכל חדר.
כל חדר מדגים אלגוריתם שונה של Reinforcement Learning, כשרמת הקושי עולה מחדר לחדר.

- חדר 1 - Value Iteration (יודעים את כל הסביבה מראש)
- חדר 2 - SARSA (לומד מניסיון, on-policy)
- חדר 3 - Q-Learning (לומד מניסיון, off-policy)
- חדר 4 - DQN (אותו רעיון של Q-Learning, אבל עם רשת נוירונים במקום טבלה)
- חדר 5 - DQN with Dynamic Obstacles and Partial Observation (The Storm)

## סיכום אלגוריתמים

| חדר | אלגוריתם | מרחב מצבים | מרחב פעולות | למה נבחר |
|-----|-----------|-------------|-------------|----------|
| 1 | Value Iteration (DP) | (שורה, עמודה, bitmask פינוקים) | 4 כיוונים | מודל ידוע — אפשר לחשב מתמטית |
| 2 | SARSA (on-policy TD) | (שורה, עמודה, כמה מפתחות) | 4 כיוונים | לומד מניסיון, זהיר — מחשיב exploration |
| 3 | Q-Learning (off-policy TD) | (שורה, עמודה, bitmask חפצים 0-7) | 4 כיוונים | לומד מה אופטימלי תיאורטית, מהיר יותר |
| 4 | DQN + Replay Buffer + Target Net | (x, y, vx, vy, wind_x, wind_y) | 9 פעולות | מרחב רציף — Q-table בלתי אפשרית |
| 5 | DQN + Partial Observation | (x, y, vx, vy) + K מכשולים קרובים | 9 פעולות | מידע חלקי + מכשולים דינמיים |

**שרת (חלון 1):**

Windows:
```bash
cd backend
python -m venv venv
./venv/Scripts/python.exe -m pip install -r requirements.txt
./venv/Scripts/python.exe main.py
```

Mac / Linux:
```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python main.py
```

**ממשק (חלון 2):**
```bash
cd frontend
npm install
npm run dev
```

פותחים את הכתובת שמודפסת (בד"כ `http://localhost:5173`). יש 5 לשוניות בראש המסך - לחיצה עוברת בין החדרים.

- **Train** - מתחיל אימון לפי הסליידרים

- **Reset** - מפה חדשה ואיפוס
- **Replay** - אחרי האימון, מאפשר לראות את הריצה הטובה ביותר צעד-צעד

## החדרים - מה בכל אחד

### חדר 1 - Value Iteration

![חדר 1](docs/screenshots/room1.png)

**הרעיון:** כשיודעים הכל על הסביבה מראש - לא צריך להתנסות בכלל. Value Iteration מחשב מתמטית מה הצעד הנכון מכל מקום בלוח.

**מה רואים:** אחרי האימון, כל תא מקבל גוון - **כהה** = רחוק מהיציאה, **כתום** = קרוב וכדאי. חצים מצביעים על הכיוון המומלץ מכל תא. גרף ה-Convergence מראה כמה איטרציות לקח עד שהאלגוריתם "הסתדר". בלוח: **תאי רוח** (כחולים) גורמים להחלקה - הסוכן יכול להחליק לכיוון אקראי. **חורים שחורים** מחזירים לנקודת ההתחלה ללא עונש. **מלכודות-חתול** עונשות בלי איפוס מיקום.

**איך זה עובד:** הוא סורק את כל הלוח שוב ושוב, מעדכן לכל תא "כמה שווה להיות פה" - ועושה את זה עד שהמספרים כמעט לא משתנים בין סריקה לסריקה.

- **מצבים:** (שורה, עמודה, bitmask - אילו פינוקים כבר נאספו)
- **תגמולים:** פינוק +5 | מלכודת-חתול −20 | חור שחור - מחזיר להתחלה | יציאה +100 | צעד −0.1
- **פרמטרים:** gamma, slip_prob, num_treats, treat_reward, trap_reward, exit_reward, theta, replay_episodes, max_steps
- **ערכים שעבדו הכי טוב:**
  - `gamma=0.99` — **מומלץ!** reward ממוצע 117.9 לעומת 108.5 עם 0.95. הלוח גדול ודרושה הסתכלות לעתיד רחוק לאסוף פינוקים ולצאת
  - `slip_prob=0.1` — אקראיות שמאתגרת בלי להפוך לבלתי ניתן לפתרון
  - `num_treats=5` — מספיק פינוקים ללמוד bitmask state בלי לעמוס
  - `treat_reward=5` קטן לעומת `exit_reward=100` — חיזקי לא ישב לנצח על פינוק
  - `trap_reward=-20` — עונש חזק שמלמד להימנע מהחתולים
  - `theta=0.0001` — סף עצירה שמבטיח התכנסות מלאה (בד"כ ~15-30 איטרציות)
  - `replay_episodes=1` — מספיק; ממוצע על יותר ריצות מייקר ללא שיפור משמעותי
  - `max_steps=200` — מספיק לסיים מסלול בלוח 10×10

### חדר 2 - SARSA

![חדר 2](docs/screenshots/room2.png)

**הרעיון:** פעם ראשונה שאנחנו לא יודעים את הסביבה מראש - חיזקי לומד מניסיון. SARSA הוא on-policy, כלומר הוא לומד בדיוק ממה שהוא עושה בפועל, כולל הטעויות.

**מה רואים:** 3 מפתחות זהב על הלוח. חיזקי חייב לאסוף אותם **בסדר מסוים** - מפתח 1, אחר כך 2, אחר כך 3 - רק אז הדלת נפתחת. ההיטמאפ בצד ימין מתעדכן בזמן ה-Replay לפי כמה מפתחות נאספו - אפשר לראות ממש איך ה"מפה המנטלית" של חיזקי משתנה.

**איך זה עובד:** אחרי כל צעד, מעדכן את הערך של מה שעשה - כולל אם בחר בפעולה אקראית. לכן SARSA זהיר יותר מ-Q-Learning.

- **מצבים:** (שורה, עמודה, כמה מפתחות כבר נאספו)
- **תגמולים:** מפתח בסדר הנכון +20 | מלכודת −15 + איפוס | יציאה +100 | צעד −0.1
- **פרמטרים:** alpha, gamma, epsilon, epsilon_decay, episodes, max_steps, K_beacons, slip_prob, key_reward, trap_reward, exit_reward, step_delay_ms
- **ערכים שעבדו הכי טוב:**
  - `alpha=0.1` — עדכון יציב, לא גורם לקפיצות ב-Q-values
  - `gamma=0.95` — מאזן בין תגמול מיידי לעתידי
  - `epsilon=0.2` עם `decay=0.995` — מספיק חקירה לגלות סדר מפתחות (600+ פרקים)
  - `episodes=500` — צריך הרבה ניסיונות לכסות צירופים (3! = 6 סדרים × מיקומים שונים)
  - `max_steps=300` — מספיק לאסוף 3 מפתחות ולהגיע ליציאה
  - `K_beacons=3` — מאזן אתגר (3! סדרים אפשריים) עם זמן אימון סביר
  - `slip_prob=0.15` — קצת גבוה יותר מחדר 1 לאתגר נוסף
  - `key_reward=20`, `trap_reward=-15`, `exit_reward=100` — יחס שמעודד איסוף מפתחות לפני יציאה

### חדר 3 - Q-Learning

![חדר 3](docs/screenshots/room3.png)

**הרעיון:** Q-Learning הוא off-policy - הוא לומד מה הפעולה הכי טובה *תיאורטית*, גם אם לא בצע אותה בפועל. זה מאפשר ניצול של חוויות ישנות ולמידה מהירה יותר.

**מה רואים:** 3 חפצים שחיזקי חייב לאסוף, אבל הפעם **בכל סדר שהוא**. X אדום נע בלוח - מגע איתו מאפס את המיקום. יש גם הוריקן סגול מסתובב שמתגלה בזמן אימון ומקפיץ קדימה - לאחר האימון הוא מסומן בהיטמאפ ובסצנה.

**איך זה עובד:** בכל עדכון, משתמש ב-`max_a Q(s',a')` - הפעולה הטובה ביותר התיאורטית, לא מה שנבחר בפועל.

- **מצבים:** (שורה, עמודה, bitmask 0-7 - אילו חפצים נאספו)
- **תגמולים:** חפץ +15 | מגע ב-X −25 + איפוס | יציאה +100 | צעד −0.1
- **פרמטרים:** alpha, gamma, epsilon, epsilon_decay, episodes, max_steps, M_fragments, x_speed, fragment_reward, x_penalty, exit_reward, step_penalty, step_delay_ms
- **ערכים שעבדו הכי טוב:**
  - `alpha=0.1`, `gamma=0.95`, `epsilon=0.2` עם `decay=0.995` — אותה אסטרטגיה כמו חדר 2
  - `episodes=500` — מרחב ה-state כאן 8× גדול מחדר 2 (2³ = 8 bitmasks)
  - `max_steps=300` — מספיק לאסוף 3 חפצים ולהגיע ליציאה תוך כדי התחמקות מה-X
  - `M_fragments=3` — 8 bitmasks (2³); בחירת 4 תיתן 16 bitmasks ותאריך אימון משמעותית
  - `x_speed=3` — X זז כל 3 צעדים: מספיק מהיר לאיום, מספיק איטי ללמוד
  - `fragment_reward=15` — תגמול ביניים שמעודד איסוף יעיל
  - `x_penalty=-25` — חזק מספיק להרתעה, לא כל כך קיצוני שחיזקי "יקפא"
  - `exit_reward=100` — ברור מה המטרה הסופית

### חדר 4 - DQN

![חדר 4](docs/screenshots/room4.png)

> **סרטון אימון - חיזקי לומד לנווט:**

https://github.com/user-attachments/assets/99a44cf6-98b4-4c64-b9fa-a5a3e34768d7

**הרעיון:** אין גריד - חיזקי שט בחלל פתוח עם מיקום ומהירות רציפים, ובנוסף **רוח אקראית** שמשתנה כל פרק ודוחפת אותו. אין טבלת Q כי יש אינסוף מצבים אפשריים, אז רשת נוירונים מחליפה אותה.

**מה רואים:** חיזקי בחלל, מנסה להגיע לעיגול למרות הרוח. בפינה ימין-עליון מוצג כיוון ועוצמת הרוח הנוכחית (למשל "↘ Wind 35%"). בהתחלה לגמרי אקראי - בהדרגה לומד לפצות על הרוח ולהגיע ליעד. גרף ה-Loss מראה כמה טוב הרשת לומדת.

**איך זה עובד:** כל 0.02 שניות (DT=0.02) הסוכן בוחר כיוון תנועה דיסקרטי ∈ {-1, 0, 1} לכל ציר. רשת נוירונים מקבלת **(x, y, vx, vy, wind_x, wind_y)** - 6 קלטים כולל הרוח (שדרוג מעל הדרישה הבסיסית של 4 קלטים) - ומחזירה ערך לכל אחת מ-9 פעולות. הרוח משתנה כל פרק, אז ה-DQN חייב ללמוד **מדיניות מותנית בהקשר** - אותה נקודה בחדר, פעולה שונה בהתאם לרוח. שני טריקים שגורמים לזה לעבוד: **Replay Buffer** ו-**Target Network**.

- **מצבים:** (x, y, vx, vy, wind_x, wind_y) - ערכים רציפים + רוח
- **תגמולים:** הגעה ליעד +100 | פגיעה בקיר −10 | כל צעד −0.05
- **פרמטרים:** learning_rate, gamma, epsilon, epsilon_decay, episodes, max_steps, batch_size, buffer_size, target_sync, exit_reward, wall_penalty, step_penalty, wind_strength
- **ערכים שעבדו הכי טוב:**
  - `learning_rate=0.001` — סטנדרטי ל-Adam optimizer, יציב ומהיר
  - `gamma=0.99` — גבוה יותר מחדרים 1-3 כי הסביבה רחבה ודרדוש לתכנן מרחוק
  - `epsilon=1.0` (התחלה) עם `decay=0.995` — חקירה מלאה בהתחלה (הרשת לא יודעת כלום)
  - `episodes=100` — מספיק עם Replay Buffer שלומד מחוויות ישנות
  - `max_steps=200` — 200 צעדים × 1 יחידה/צעד = מספיק לחצות 10×10 מ'
  - `batch_size=64` — מאזן מהירות למידה ויציבות
  - `buffer_size=10000` — 10K חוויות מספיקות לריסוק קורלציות
  - `target_sync=100` — עדכון Target Network כל 100 צעדים: מספיק יציב
  - `wind_strength=0.4` — מאתגר (40% ממהירות מקסימלית) בלי להיות בלתי אפשרי
  - `step_penalty=-0.05` — עונש קטן שדוחף למהירות בלי להכביד

### חדר 5 - DQN with Dynamic Obstacles and Partial Observation (The Storm)

![חדר 5](docs/screenshots/room5.png)

> **סרטון אימון - חיזקי מתמודד עם מכשולים:**

https://github.com/user-attachments/assets/5cc7f55e-04a9-4b19-ac14-15c502a80647

**הרעיון:** כמו חדר 4, רק שעכשיו יש מכשולים שזזים ומיקומם משתנה בכל ניסיון. חיזקי לא יכול לשנן מפה - חייב ללמוד לזהות מכשולים ולהגיב.

**מה רואים:** מכשולים אדומים נעים בחלל. חיזקי רואה רק את K המכשולים הקרובים ביותר אליו (observation חלקית). אחרי האימון - כפתור "Test on new layout" בודק על 10 מפות שמעולם לא ראה ומדווח כמה פעמים הצליח.

**איך זה עובד:** ה-state כולל את המרחק למכשולים הקרובים. הוספנו גם Potential-based shaping - תגמול קטן על כל צעד לכיוון היציאה - כדי שחיזקי לא "יקפא" מפחד.

- **מצבים:** (x, y, vx, vy) + מרחקים ל-K מכשולים הקרובים ביותר
- **תגמולים:** יציאה +100 | פגיעה במכשול −20 | פגיעה בקיר −10
- **פרמטרים:** learning_rate, gamma, epsilon, epsilon_decay, episodes, max_steps, batch_size, buffer_size, target_sync, n_obstacles, k_visible, visibility_range, obstacle_drift, drag, exit_reward, wall_penalty, obstacle_penalty, step_penalty, shaping_coef
- **ערכים שעבדו הכי טוב:**
  - **פרמטרי רשת** (זהים לחדר 4): `learning_rate=0.001`, `gamma=0.99`, `epsilon=1.0` + `decay=0.995`, `batch_size=64`, `buffer_size=10000`, `target_sync=100`
  - `episodes=300` — יותר מחדר 4 כי כל פרק יש מכשולים חדשים שמצריכים הכללה
  - `max_steps=500` — יותר צעדים לפרק כי הניווט קשה עם מכשולים
  - `n_obstacles=5` — מאתגר אבל עביר; 10 מכשולים הופכים את זה לבלתי אפשרי
  - `k_visible=3` — רואה 3 מכשולים קרובים: מספיק מידע בלי להעמיס על הרשת
  - `visibility_range=3.0` מ' — מרחק ראייה שמאפשר תגובה בזמן
  - `obstacle_drift=0.01` — תנועה איטית שמאפשרת לחיזקי להסתגל
  - `drag=0.85` — אינרציה שמוסיפה ריאליזם לפיזיקה
  - `obstacle_penalty=-20` — עונש חמור שמלמד להתחמק
  - `shaping_coef=8` — potential-based reward shaping: בונוס קטן על קדמה ליציאה שמונע "קפיאה" מפחד
  - `step_penalty=-0.05` — דוחף ליעילות בלי להכביד
