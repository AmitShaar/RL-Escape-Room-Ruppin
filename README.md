# פרויקט גמר: חדרי בריחה - למידה עם חיזוקים (Hizki In Space RL)

## מבוא

בפרויקט זה מוצגים שישה "חדרים", כאשר בכל חדר מיושם אלגוריתם שונה של Reinforcement Learning. הסוכן הוא **חיזקי** - כלב בחליפת חלל שמחפש עצם בכל חדר.

- חדר 1 - Value Iteration (Dynamic Programming, Model-Based)
- חדר 2 - SARSA (On-Policy, Model-Free)
- חדר 3 - Q-Learning (Off-Policy, Model-Free)
- חדר 4 - DQN בסביבה רציפה עם מהירות דיסקרטית (Deep Q-Network, Off-Policy)
- חדר 5 - DQN + מכשולים דינמיים + Partial Observation (The Storm)
- חדר 6 - Curriculum Learning + Q-Learning (גריד שגדל בהדרגה)

## הוראות הרצה

הפרויקט מורכב משרת (backend בפייתון) ולקוח (frontend בדפדפן) - יש להריץ את שניהם במקביל.

**שרת:**
```bash
cd backend
python -m venv venv
./venv/Scripts/python.exe -m pip install -r requirements.txt
./venv/Scripts/python.exe main.py
```

**לקוח (בחלון טרמינל נוסף):**
```bash
cd frontend
npm install
npm run dev
```

לאחר מכן פותחים את הכתובת שמודפסת בטרמינל (בד"כ `http://localhost:5173`) בדפדפן. בראש המסך יש 6 לשוניות, אחת לכל חדר.

בכל חדר יש כפתורים:
- **Train** - מתחיל אימון לפי הפרמטרים שנבחרו בסליידרים
- **Pause / Resume** - להקפיא ולהמשיך אימון שכבר רץ
- **Reset** - לאפס את החדר (כולל מפה חדשה, אם רלוונטי)
- אחרי שהאימון מסתיים, יש **Replay** - סרגל שמשחק את הריצה הטובה ביותר, צעד-צעד

## תיאור החדרים

### חדר 1 - Value Iteration

![חדר 1 - Value Iteration](docs/screenshots/room1.png)

**מטרה:** להראות שכשהמודל ידוע מראש, אפשר לחשב את המדיניות האופטימלית מתמטית — בלי להתנסות בכלל.

**מה רואים:** לאחר האימון, כל תא בלוח מקבל גוון — **כהה = ערך נמוך** (רחוק מהיציאה), **כתום = ערך גבוה** (קרוב/שווה). חצים מראים לאן כדאי ללכת מכל תא. ניתן לגרור את סרגל ה-Replay ולראות את הריצה הטובה ביותר צעד-צעד.

**מה האלגוריתם עושה:** סורק את כל הלוח שוב ושוב, מחשב לכל תא "כמה שווה להיות כאן" עד שהערכים מתייצבים. גרף ה-Convergence מראה כמה "קפיצות" ערכיות היו בכל סריקה — ככל שמתקרב לאפס, האלגוריתם סיים.

- **מצבים:** (שורה, עמודה, אילו פינוקים נאספו) | **תגמולים:** פינוק +5, מלכודת −20, יציאה +100, צעד −0.1
- **פרמטרים:** gamma, slip_prob, treat_reward, trap_reward, bone_reward, replay_episodes, max_steps
- **ערכים אופטימליים שנמצאו:** `gamma=0.95` — מספיק חשיבה לטווח רחוק לאסוף פינוקים בדרך ליציאה, בלי להאט את ההתכנסות. `slip_prob=0.1` — אקראיות מספקת לאתגר אך לא משתקת. `treat_reward=5` קטן יחסית ל-`bone_reward=100` — מבטיח שחיזקי לא "יתנדנד" על פינוקים לנצח. `replay_episodes=10` נותן מדגם מספיק גדול לבחירת הריצה הטובה.

### חדר 2 - SARSA

![חדר 2 - SARSA](docs/screenshots/room2.png)

**מטרה:** להראות למידת TD on-policy — הסוכן לומד תוך כדי ניסוי וטעייה, כשהמודל לא ידוע.

**מה רואים:** 3 מפתחות זהב על הלוח שחיזקי חייב לאסוף **בסדר מסוים** לפני שהיציאה נפתחת. ההיטמאפ מתעדכן בזמן ה-Replay לפי מספר המפתחות שנאספו — כשאספת הכל, פתאום האזור ליד היציאה "מדליק" ומראה שהוא שווה.

**מה האלגוריתם עושה:** לומד מה שווה על ידי ניסיון בפועל. בניגוד לחדר 1 — אין ידע מוקדם, רק ניסיון. ה-on-policy אומר שהוא לומד בדיוק את ההתנהגות שהוא מבצע (כולל הטעויות).

- **מצבים:** (שורה, עמודה, כמה מפתחות נאספו) | **תגמולים:** מפתח +20, מלכודת −15, יציאה +100
- **פרמטרים:** alpha, gamma, epsilon, episodes, max_steps
- **ערכים אופטימליים שנמצאו:** `alpha=0.1` — קצב למידה יציב שלא גורם לתנודות בQ-values. `gamma=0.95` — מספיק לתכנון מסלול עם 3 מפתחות ברצף. `epsilon=0.2` עם `decay=0.995` — מבטיח מספיק חקירה בהתחלה לגלות את סדר המפתחות. `episodes=500` — הכרחי לכיסוי מרחב ה-state עם 4 ערכי visited_count.

### חדר 3 - Q-Learning

![חדר 3 - Q-Learning](docs/screenshots/room3.png)

**מטרה:** להדגים Q-Learning (off-policy TD) בסביבה עם **אובייקטים דינמיים** — X חוסם נע, פורטל אקראי, וחפצים לאיסוף ללא סדר קבוע.

**מה רואים:** 3 חפצים (יהלומים) שחיזקי חייב לאסוף **בכל סדר שהוא** לפני שהיציאה נפתחת. X אדום נע לאורך מסלול קבוע — מגע איתו מאפס את המיקום. פורטל סגול מסמן קיצור דרך שמתגלה בזמן חקירה ומוצג בהיטמאפ אחרי האימון.

**מה האלגוריתם עושה:** Q-Learning לומד את הפעולה הטובה **תיאורטית** (max_a Q), גם אם לא ביצע אותה בפועל. off-policy = הלמידה מנותקת מהמדיניות המבוצעת — ניתן לנצל חוויות ישנות ולהכליל טוב יותר.

- **מצבים:** (שורה, עמודה, bitmask של חפצים 0-7) | **תגמולים:** חפץ +15, מגע ב-X −25, יציאה +100, צעד −0.1
- **פרמטרים:** alpha, gamma, epsilon, episodes, max_steps
- **ערכים אופטימליים שנמצאו:** `alpha=0.1`, `gamma=0.95`, `epsilon=0.2` — כמו חדר 2 אך המרחב כפול (8 bitmasks במקום 4). `episodes=500` — נדרש לכסות את כל 8 המצבים ואת תנועת ה-X. עונש X של `−25` חזק מספיק להרתעה אך לא חוסם לגמרי — האלגוריתם לומד לחכות לפרצה.

### חדר 4 - DQN

![חדר 4 - DQN](docs/screenshots/room4.png)

> **הדגמת אימון — חיזקי לומד לנווט בסביבה רציפה:**

https://github.com/user-attachments/assets/c23db440-875b-4eb3-86d0-3b98435a40aa

**מטרה:** לעבור מסביבה דיסקרטית (גריד) לסביבה **רציפה** שבה אין טבלה — רשת נוירונים מחליפה אותה.

**מה רואים:** חיזקי שט בחלל פתוח ומנסה להגיע לעיגול הירוק. בהתחלה הוא נע אקראית לחלוטין — בהדרגה לומד לכוון לכיוון הנכון. גרף ה-Loss מראה שהרשת משתפרת.

**מה האלגוריתם עושה:** במקום טבלה, רשת נוירונים מקבלת (x, y, vx, vy) ומחזירה ערך לכל אחת מ-9 הפעולות. Replay Buffer מאחסן חוויות ומדגם מהן באקראי כדי ללמוד יציב.

- **מצבים:** (x, y, vx, vy) רציפים | **תגמולים:** יציאה +100, קיר −10, צעד −0.05
- **פרמטרים:** learning rate, gamma, epsilon, decay, episodes, max_steps
- **ערכים אופטימליים שנמצאו:** `gamma=0.99` — גבוה במיוחד כי המרחב רחב ונדרשת חשיבה ל-~11 צעדים קדימה. `lr=0.001` — קצב סטנדרטי ל-Adam, יציב על הרשת. `epsilon=1.0` עם `decay=0.995` — מתחיל אקראי לחלוטין (הרשת לא יודעת כלום), יורד לאט מספיק. `episodes=100` מספיקות בזכות המרחב הפשוט יחסית.

### חדר 5 - DQN + מכשולים דינמיים (The Storm)

![חדר 5 - The Storm](docs/screenshots/room5.png)

**מטרה:** הקשה ביותר — כמו חדר 4, אבל עם מכשולים שזזים ומיקומם משתנה בכל ניסיון. הסוכן לא יכול לשנן מפה — חייב ללמוד להגיב.

**מה רואים:** מכשולים אדומים נעים בחלל. חיזקי רואה רק את המכשולים הקרובים אליו (partial observation). אחרי האימון — כפתור "Test on new layout" בודק אם מה שנלמד עובד על מפות חדשות לגמרי.

**מה האלגוריתם עושה:** State כולל גם offset למכשולים הקרובים. Potential-based shaping מוסיף תגמול קטן על כל צעד לכיוון היציאה כדי שהסוכן לא "יקפא" מפחד המכשולים.

- **מצבים:** (x, y, vx, vy) + מרחקים ל-K מכשולים קרובים | **תגמולים:** יציאה +100, מכשול −20, קיר −10
- **פרמטרים:** learning rate, gamma, epsilon, N obstacles, visibility range
- **ערכים אופטימליים שנמצאו:** `N_obstacles=5` — מספיק מכשולים לאתגר אך לא בלתי עביר. `visibility_range=3.0` מ' — מספיק לראות מכשול ולהגיב בזמן במהירות 1 מ'/צעד. `episodes=300` — יותר מחדר 4 כי המכשולים אקראיים ונדרש לימוד הכללה. עונש מכשול `−20` גדול מעונש קיר — מלמד שמכשול מסוכן יותר מגבול.

### חדר 6 - Curriculum Learning

![חדר 6 - Curriculum Learning](docs/screenshots/room6.png)

**מטרה:** להראות שאימון הדרגתי (קל → קשה) מהיר יותר מאשר אימון ישיר על הבעיה הקשה.

**מה רואים:** הלוח גדל בשלבים — 4×4 ראשון, אחר כך 6×6, לבסוף 10×10. גרף התגמול מראה **שלושה שלבים ברורים** — עלייה, ירידה קצרה כשהלוח גדל, עלייה שנייה מהירה יותר. ההיטמאפ מתעדכן בכל שלב לגודל הנוכחי.

**מה האלגוריתם עושה:** Q-Learning רגיל, אבל בסיום כל שלב טבלת ה-Q מועתקת לתוך טבלה גדולה יותר. הידע על הפינה הקטנה "נשמר" ומוסיף יתרון על השלב הבא.

- **מצבים:** (שורה, עמודה) ברשת משתנה | **תגמולים:** יציאה +100, צעד −0.1
- **פרמטרים:** alpha, gamma, epsilon, step_delay_ms
- **ערכים אופטימליים שנמצאו:** `epsilon=0.3` התחלתי עם `decay=0.99` — גבוה יותר משאר החדרים כי הגריד גדל ונדרש לחקור שוב אזורים חדשים. `alpha=0.1`, `gamma=0.95` — סטנדרטי ויציב. חלוקת אפיזודות: 100+150+250 = 500 סה"כ — כל שלב מקבל יותר אפיזודות ככל שהגריד גדל ומורכב יותר.
